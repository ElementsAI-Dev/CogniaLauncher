# 国际化参考

CogniaLauncher 使用 `next-intl` 实现国际化，支持中英文双语。

---

## 概览

| 指标 | 数值 |
|------|------|
| 支持语言 | 中文 (zh)、英文 (en) |
| 键值数量 | 1640+ |
| 翻译文件 | `messages/en.json`、`messages/zh.json` |

---

## 使用方式

### 组件中使用

```tsx
import { useTranslations } from "next-intl";

function MyComponent() {
  const t = useTranslations("settings");

  return <h1>{t("title")}</h1>;
}
```

### 嵌套键

翻译文件使用嵌套 JSON 结构：

```json
{
  "settings": {
    "title": "设置",
    "appearance": {
      "title": "外观",
      "theme": "主题",
      "accentColor": "强调色"
    }
  }
}
```

访问：`t("appearance.title")` → "外观"

---

## 键名规范

| 前缀 | 用途 | 示例 |
|------|------|------|
| `dashboard.*` | 仪表板 | `dashboard.title` |
| `environments.*` | 环境管理 | `environments.install` |
| `packages.*` | 包管理 | `packages.search` |
| `settings.*` | 设置 | `settings.appearance.theme` |
| `downloads.*` | 下载管理 | `downloads.pause` |
| `cache.*` | 缓存管理 | `cache.clean` |
| `wsl.*` | WSL 管理 | `wsl.listDistros` |
| `logs.*` | 日志 | `logs.filter` |
| `about.*` | 关于 | `about.version` |
| `common.*` | 通用 | `common.cancel` |

---

## 添加新翻译

1. 同时在 `messages/en.json` 和 `messages/zh.json` 中添加
2. 保持两个文件的键结构完全一致
3. 使用有意义的嵌套路径
4. 提交前验证两文件键数一致

---

## 系统托盘

系统托盘菜单也支持多语言，通过后端读取当前语言设置动态生成菜单文本。
