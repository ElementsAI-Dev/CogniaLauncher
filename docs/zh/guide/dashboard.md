# 仪表板

仪表板是 CogniaLauncher 的主页面，提供系统状态概览与可完整自定义的首页布局。

---

## 功能概述

### 环境概览

仪表板顶部展示已安装的运行时环境状态：

- **已安装环境数量** — Node.js、Python、Rust 等
- **当前激活版本** — 全局正在使用的版本
- **可用更新** — 检测到的版本更新

### 快速操作

- 一键安装常用环境
- 快速切换版本
- 打开设置页面

### 完整布局自定义

首页自定义由两个联动入口组成：

- **Header 编辑模式** — 在网格内启用拖拽排序、尺寸调整、显隐切换、移除操作。
- **自定义对话框** — 管理组件目录、分类过滤、添加操作与恢复默认。

行为保障：

- **统一策略来源** — 添加/移除/显隐都基于同一组件注册策略（`allowMultiple`、`required`、`defaultVisible`）。
- **受策略约束的添加** — 达到实例上限的组件在对话框中会显示为不可继续添加。
- **确定性恢复默认** — 恢复默认始终回到同一套标准组件集合与顺序。
- **持久化与迁移安全** — 布局恢复/迁移时会归一化异常数据（非法尺寸、重复 ID、未知组件、无效载荷回退）。

### 首页概览与反馈

首页在组件网格上方新增了工作台状态条：

- **概览就绪摘要** — 快速展示环境、软件包、系统三个分区当前是已就绪、加载中还是需要关注。
- **分层加载** — 已经准备好的组件会继续可用，不会因为其他分区仍在加载而整体阻塞。
- **可执行的降级态** — 出问题的组件会在本地直接提供重试或深链入口，而不只依赖页面级错误横幅。

关键组件行为：

- **欢迎 / 工作台就绪卡片** — 新用户看到引导步骤；已经配置完成的用户会看到简化后的“工作台已准备就绪”卡片，并可直接进入环境、软件包和设置页面。
- **环境 / 软件包列表** — 空态会直接提供下一步操作；加载或错误时会在组件内部保留恢复路径。
- **健康检查 / 更新 / 系统信息** — 在无数据或降级数据场景下，组件会明确提示“运行检查”“立即检查”或“重试”等下一步动作。

### 默认布局升级

- **首次进入和恢复默认** — 新用户和“恢复默认”路径会使用最新的首页标准排序，让概览和快捷动作优先于诊断类与明细类组件。
- **现有自定义布局保持不变** — 升级 CogniaLauncher 不会静默覆盖用户已有的首页自定义布局；只有首次布局或手动恢复默认才会应用新的标准首页体验。

### 系统状态

- Provider 可用性检查
- 缓存使用情况统计
- 网络连接状态
