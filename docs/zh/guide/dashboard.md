# 仪表板

仪表板是 CogniaLauncher 的主页面，提供系统状态概览与可完整自定义的首页布局。

---

## 功能概述

### 环境概览

仪表板顶部展示已安装的运行时环境状态：

- **已安装环境数量** — Node.js、Python、Rust 等
- **当前激活版本** — 全局正在使用的版本
- **可用更新** — 检测到的版本更新

### 快速操作

- 一键安装常用环境
- 快速切换版本
- 打开设置页面

### 完整布局自定义

首页自定义由两个联动入口组成：

- **Header 编辑模式** — 在网格内启用拖拽排序、尺寸调整、显隐切换、移除操作。
- **组件设置工具条** — 新的 insight widgets 可以直接在编辑工具条里调整范围、视图模式、分组、共享范围覆盖和条目数量等轻量设置。
- **自定义对话框** — 管理组件目录、分类过滤、添加操作、可配置组件标记、分析能力标记与恢复默认。

行为保障：

- **统一策略来源** — 添加/移除/显隐都基于同一组件注册策略（`allowMultiple`、`required`、`defaultVisible`）。
- **Registry 驱动的组件设置** — 可配置的 insight widgets 会从 dashboard widget registry 中读取默认设置，并支持一键恢复。
- **受策略约束的添加** — 达到实例上限的组件在对话框中会显示为不可继续添加。
- **确定性恢复默认** — 恢复默认始终回到同一套标准组件集合与顺序。
- **持久化与迁移安全** — 布局恢复/迁移时会归一化异常数据（非法尺寸、重复 ID、未知组件、无效载荷回退，以及非法组件设置回退到默认值）。

### Dashboard Insight 工作台

首页现在增加了一层新的 insight 聚合能力，把健康检查、下载、安装历史和最近工具使用等信号整理为首页可直接消费的视图：

- **关注中心** — 在首页优先展示最值得处理的问题，并直接深链到对应功能页。
- **共享范围切换** — 首页 Header 可以把兼容的可视化组件在 `7d` 和 `30d` 观察窗口之间统一切换。
- **工作区趋势** — 既支持单指标趋势，也支持在同一视图里对比安装、下载和更新三条序列。
- **Provider 健康矩阵** — 同时支持紧凑状态列表和热力矩阵式扫描视图。
- **活动概览** — 原有分布概览现在可以切换到活动强度视图，用来总结下载、包管理和工具使用的集中时段。
- **最近活动流** — 把软件包、下载和工具箱活动合并成统一时间流。

Insight 数据采用按需加载：

- **按可见组件取数** — 只有当前可见 widget 依赖的二级数据源才会触发加载。
- **保留首页原有分阶段加载** — 环境 / 软件包 / 设置的首页启动节奏不变，insight 层按需附加。
- **安全降级** — 单个 insight widget 可以独立进入空态或降级态，而不会阻塞整个首页。

### 首页概览与反馈

首页在组件网格上方新增了工作台状态条：

- **概览就绪摘要** — 快速展示环境、软件包、系统三个分区当前是已就绪、加载中还是需要关注。
- **分层加载** — 已经准备好的组件会继续可用，不会因为其他分区仍在加载而整体阻塞。
- **可执行的降级态** — 出问题的组件会在本地直接提供重试或深链入口，而不只依赖页面级错误横幅。

关键组件行为：

- **欢迎 / 工作台就绪卡片** — 新用户看到引导步骤；已经配置完成的用户会看到简化后的“工作台已准备就绪”卡片，并可直接进入环境、软件包和设置页面。
- **环境 / 软件包列表** — 空态会直接提供下一步操作；加载或错误时会在组件内部保留恢复路径。
- **健康检查 / 更新 / 系统信息** — 在无数据或降级数据场景下，组件会明确提示“运行检查”“立即检查”或“重试”等下一步动作。

### 默认布局升级

- **首次进入和恢复默认** — 新用户和“恢复默认”路径会使用最新的首页标准排序，并把新的 insight widgets 一并放入标准首页布局。
- **现有自定义布局保持不变** — 升级 CogniaLauncher 不会静默覆盖用户已有的首页自定义布局；只有首次布局或手动恢复默认才会应用新的标准首页体验。

### 系统状态

- Provider 可用性检查
- 缓存使用情况统计
- 网络连接状态

## 状态管理

仪表板布局数据存储在 `lib/stores/dashboard.ts`（Zustand）：

- 小部件顺序、尺寸与可见性
- 共享的 Dashboard 可视分析上下文，例如当前的范围选择
- 可配置 insight widgets 的 widget-specific settings
- Store/UI 共享的组件实例策略 helper
- 持久化布局迁移与归一化
