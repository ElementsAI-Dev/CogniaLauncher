# 缓存管理

CogniaLauncher 使用 SQLite + JSON 双后端缓存系统，同时把默认下载目录、外部工具缓存和用户自定义缓存目录纳入统一的维护工作流。

---

## 功能概述

### 缓存类型

| 类型 | 存储方式 | 用途 |
|------|----------|------|
| 下载缓存 | 文件系统 | 已下载的包和二进制文件 |
| 元数据缓存 | SQLite | 包信息、版本列表、搜索结果 |
| 默认下载目录 | 文件系统 | 系统下载目录中由 Cognia 托管的文件 |
| 外部工具缓存 | 文件系统 / 工具命令 | npm、pip、uv、Docker 等外部工具缓存 |
| 自定义缓存目录 | 用户指定路径 | 用户手动登记、希望统一纳入维护的缓存目录 |
| 版本缓存 | JSON | 前端版本检测结果缓存 |

### 缓存统计

在 **缓存** 页面可查看：

- 缓存总大小
- 条目数量
- 命中率
- 最后清理时间

---

## 缓存操作

### 清理缓存

按缓存类型清理：

- **全部清理** — 删除所有缓存
- **按类型清理** — 仅清理下载缓存或元数据缓存
- **默认下载目录清理** — 只影响位于系统下载目录下、由 Cognia 托管且处于安全边界内的文件
- **按 Provider 清理** — 清理特定 Provider 的缓存
- **清理过期缓存** — 先预览过期条目，再确认清理
- **自定义缓存目录清理** — 对已登记的自定义缓存路径执行统一检测与清理

### Drilldown 与详情页

- `/cache/download` — 下载缓存详情
- `/cache/metadata` — 元数据缓存详情
- `/cache/default_downloads` — 默认下载目录详情，展示下载根目录、安全边界、候选文件与跳过原因
- `/cache/external` — 外部缓存与自定义缓存详情页；可通过 query target 自动定位到指定 Provider 或自定义缓存项

常见入口包括：

- 缓存概览卡片和监控摘要中的 **查看详情**
- 自定义缓存目录列表中的 **查看详情**
- 外部缓存清理结果中的 Provider 链接

### 浏览缓存条目

缓存条目浏览器支持：

- 按键或路径搜索
- 按创建时间、大小、命中次数排序
- 类型筛选：**全部**、**下载**、**元数据**、**部分**、**索引**
- 批量删除，并可选择移动到回收站

### 验证缓存

检查缓存完整性：

- 校验文件哈希
- 检测损坏的条目
- 报告不一致项

### 修复缓存

自动修复发现的问题：

- 删除损坏条目
- 重建索引
- 清理孤立文件

## 缓存设置

| 设置 | 描述 | 默认值 |
|------|------|--------|
| 最大缓存大小 | 缓存占用上限 | 无限制 |
| 最大缓存天数 | 缓存过期天数 | 无限制 |
| 自动清理 | 超过阈值自动清理 | 关闭 |
| 清理阈值 | 触发自动清理的占用百分比 | 80% |
| 监控间隔 | 缓存状态检查间隔（秒） | 300 |
| 外部缓存监控 | 监控外部包管理器的缓存 | 关闭 |
| 排除的外部 Provider | 在检测和清理中忽略指定 Provider | 空 |
| 自定义缓存目录 | 将指定目录作为统一缓存范围参与检测与清理 | 空 |

---

## 缓存迁移

支持将缓存数据在不同存储后端之间迁移（SQLite ↔ JSON / 新路径）。

通过缓存页面的 **迁移** 按钮操作。
