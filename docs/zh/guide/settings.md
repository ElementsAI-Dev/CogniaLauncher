# 设置与主题

CogniaLauncher 提供丰富的设置选项和主题定制功能。

---

## 设置页面

通过侧边栏底部的齿轮图标进入设置页面。设置按类别分组：

### 常规设置

- 语言切换（中/英）
- 并行下载数
- 版本解析策略
- 元数据缓存 TTL
- 自动更新元数据

### 网络设置

- 请求超时时间
- 失败重试次数
- HTTP/HTTPS 代理

### 路径设置

- CogniaLauncher 数据目录
- 缓存目录
- 环境安装目录
- 支持文件夹选择器（Tauri 模式）

### 缓存设置

- 最大缓存大小
- 缓存过期天数
- 自动清理开关及阈值
- 缓存监控间隔
- 外部缓存监控

### 外观设置

- 主题模式（浅色/深色/跟随系统）
- 强调色
- 图表颜色主题（6 种方案）
- 界面密度与圆角半径
- 减少动画、原生窗口效果、背景图细节调节
- 外观预设工作区（保存、重命名、应用、删除预设）

### 安全设置

- 允许 HTTP（仅开发环境）
- 证书验证
- 自签名证书

### 引导控制

设置页提供了独立的 **引导与导览** 卡片，用于管理首次启动体验：

- 从上次步骤继续已暂停的设置向导
- 从模式选择重新运行引导（快速/详细）
- 在完成设置后单独启动引导导览
- 查看引导状态（已完成 / 已跳过 / 已暂停 / 可继续）
- 管理上下文气泡提示（重置、全部关闭、启用/禁用）

### 保存、导入与重置可靠性

设置中心现在使用显式的基线/草稿/保存快照模型，确保在后台刷新配置时本地编辑不被隐式覆盖。

- 当后端配置刷新且存在未保存草稿时，CogniaLauncher 会保留草稿并提示冲突
- 导入流程升级为四阶段：解析、校验、差异预览、显式确认
- 保存会按键记录结果，失败项会保留在草稿中并支持“仅重试失败项”
- 分区重置只影响该分区键；全局重置会统一同步配置、应用设置与外观状态

### 终端配置流程可靠性

终端页面在 Shell 配置与配置文件管理方面新增了更严格的保护与刷新规则。

- 切换标签页或切换 Shell/配置文件目标时，若存在未保存草稿会先弹出确认
- 配置文件导入先执行预校验并展示摘要（有效/冲突/无效计数），确认后才会执行
- 导入冲突处理策略改为显式选择：先选 **合并** 或 **替换**，再执行导入
- 代理、环境变量、PowerShell 相关标签页按资源失效状态按需刷新，避免无效重复加载
- PowerShell 模块管理现在会在当前终端工作流中保留库搜索、安装/更新/卸载反馈，以及执行策略应用结果
- 模板管理现已支持自定义模板创建，Shell/框架卡片也提供详情下钻，方便在执行缓存清理等高风险操作前先完成检查

## 主题系统

### 颜色方案

基于 CSS 变量实现，定义在 `app/globals.css` 中。

支持的主题模式：

- **浅色模式** — 明亮背景
- **深色模式** — 暗色背景
- **跟随系统** — 自动匹配操作系统主题

### 强调色

可选择不同的强调色方案来个性化界面。

### 自定义工作区

外观分组中新增了统一的 **界面自定义工作区**：

- 一键选择并应用已保存预设
- 将当前外观保存为新预设
- 重命名或删除非默认预设
- 仅重置外观字段，不影响其他分组的待保存变更

应用预设时会统一更新受外观控制的字段（主题、强调色、图表主题、圆角、密度、动效、背景选项、窗口效果）。

### 预设迁移兼容

从不包含预设元数据的旧版本升级时，CogniaLauncher 会根据现有外观值自动生成兼容的默认预设。  
导入旧格式（无预设字段）的设置文件时，也会自动补齐默认预设结构。

### 图表颜色主题

数据可视化图表支持 6 种配色方案：

| 主题 | 描述 |
|------|------|
| default | 默认配色 |
| vibrant | 鲜艳配色 |
| pastel | 柔和配色 |
| ocean | 海洋蓝配色 |
| sunset | 日落暖色 |
| monochrome | 单色配色 |

## 设置搜索

设置页面内置搜索功能：

- 按关键词过滤设置项
- 支持中英文搜索
- 自动定位到匹配的设置区域
- IntersectionObserver 追踪当前活动区域

## 状态管理

设置数据存储在多个 Zustand Store 中：

| Store | 用途 |
|-------|------|
| `appearance.ts` | 主题、强调色、图表颜色主题 |
| `settings.ts` | 通用设置 |
| `window-state.ts` | 窗口状态 |

所有 Store 使用 `persist` 中间件实现 localStorage 持久化。
