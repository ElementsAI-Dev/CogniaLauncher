# Internationalization Reference

CogniaLauncher uses `next-intl` for internationalization, supporting Chinese and English bilingual.

---

## Overview

| Metric | Value |
|--------|-------|
| Supported Languages | Chinese (zh), English (en) |
| Key Count | 1640+ |
| Translation Files | `messages/en.json`, `messages/zh.json` |

---

## Usage

### In Components

```tsx
import { useTranslations } from "next-intl";

function MyComponent() {
  const t = useTranslations("settings");

  return <h1>{t("title")}</h1>;
}
```

### Nested Keys

Translation files use nested JSON structure:

```json
{
  "settings": {
    "title": "Settings",
    "appearance": {
      "title": "Appearance",
      "theme": "Theme",
      "accentColor": "Accent Color"
    }
  }
}
```

## Key Naming Conventions

| Prefix | Purpose | Example |
|--------|---------|---------|
| `dashboard.*` | Dashboard | `dashboard.title` |
| `environments.*` | Environment management | `environments.install` |
| `packages.*` | Package management | `packages.search` |
| `settings.*` | Settings | `settings.appearance.theme` |
| `downloads.*` | Download management | `downloads.pause` |
| `cache.*` | Cache management | `cache.clean` |
| `wsl.*` | WSL management | `wsl.listDistros` |
| `logs.*` | Logs | `logs.filter` |
| `about.*` | About | `about.version` |
| `common.*` | Common | `common.cancel` |

---

## Adding New Translations

1. Add simultaneously to both `messages/en.json` and `messages/zh.json`
2. Keep the key structure of both files perfectly consistent
3. Use meaningful nested paths
4. Verify both files have equal key counts before committing

---

## System Tray

The system tray menu also supports multiple languages, dynamically generating menu text by reading the current language setting from the backend.
